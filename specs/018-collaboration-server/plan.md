# Implementation Plan: Collaboration Server

**Branch**: `018-collaboration-server` | **Date**: 2026-06-07 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from `specs/018-collaboration-server/spec.md`

## Summary

Phase 8 delivers the standalone Hocuspocus collaboration server (`apps/collab`) with per-document rooms, session-based auth hook, Yjs state persistence + periodic file write-back, admin-configurable flush interval, user awareness broadcast, upload/delete blocking while a room is active, and a graceful-shutdown flush. It also refactors the existing save, delete, and load file use cases to be collaboration-aware. User input: "must refactor existing logic to save, delete, create, and load files".

## Technical Context

**Language/Version**: TypeScript 6, Node.js 24. New `apps/collab` package: `"type": "commonjs"` (matching `apps/api`).

**Primary Dependencies**: `@hocuspocus/server` (WebSocket + Yjs sync); `yjs` (CJS build via `createRequire` вЂ” same workaround as existing stub); `pino` (logging); `@asciidocollab/domain`; `@asciidocollab/infrastructure`; `@asciidocollab/db`.

**Storage**: PostgreSQL via Prisma (new `CollaborationSession` model for active room tracking; existing `SystemSetting` for write-back interval). Filesystem: `FilesystemYjsStateStore` (existing) + `FilesystemProjectFileStore` (existing) вЂ” both reused via domain port interfaces.

**Testing**: Jest + in-memory fakes for domain unit tests; testcontainers for `PrismaCollaborationSessionRepository` integration test.

**Target Platform**: Linux server process, standalone вЂ” `pnpm --filter @asciidocollab/collab start`.

**Project Type**: WebSocket server (real-time collaboration).

**Performance Goals**: <1 s sync latency (SC-001); <2 s join time (SC-002); 50+ concurrent rooms (SC-006).

**Constraints**: Domain layer zero external deps; TDD red-green-refactor; `Result<T,E>` for all fallible domain ops; no `as` casts; no `console.log`; JSDoc on all public exports; test files under `tests/` root only.

> **Post-analysis notes** (speckit-analyze 2026-06-08):
> - **FR-012 deferred to Phase 9**: FR-012 (editor read-only when collaboration server is unreachable) requires the CodeMirror-Yjs provider binding, which is out of scope for Phase 8. It has been marked as deferred in `spec.md` and carries no tasks in this phase.
> - **TDD task ordering**: `tasks.md` has been reordered so that every test task precedes the production code task it covers (Constitution Principle II). Implementation tasks reference their corresponding RED test task explicitly.
> - **apps/api composition root wiring (T036)**: An explicit task has been added to inject `PrismaCollaborationSessionRepository` into `SaveDocumentContentUseCase` and `DeleteFileUseCase` at the `apps/api` composition root; without it, the active-session guard (FR-011) would silently remain inactive in production.

> **Post-review additions** (security & architecture review 2026-06-07):
> - `apps/api/src/internal-server.ts` вЂ” second Fastify instance on loopback; enforces network-level isolation of the auth endpoint.
> - `ASCIIDOCOLLAB_COLLAB_AUTH_TIMEOUT_MS` вЂ” new env var; defaults to 3000 ms; prevents auth hook hang.
> - `ASCIIDOCOLLAB_COLLAB_INTERNAL_PORT` вЂ” new env var in both `apps/api` and `apps/collab` configs; defaults to 4001.
> - `CollaborationSession` FK to `Document` with `onDelete: Cascade` вЂ” prevents orphan session records.
> - Ordered graceful shutdown sequence documented in research Decision 5.

## Implementation Sequence

Tasks are defined in `tasks.md` (generated by `/speckit-tasks`). The logical order is:

1. **DB schema** вЂ” Add `CollaborationSession` model to Prisma schema (ask user before migrating).
2. **Domain layer**
   a. New port: `CollaborationSessionRepository` (in `ports/project/`, not `ports/storage/`)
   b. New error: `ActiveCollaborationSessionError`
   c. Add `DocumentRepository.findByYjsStateId`
   d. In-memory fakes for all new ports (extended)
   e. New use cases: `OpenCollaborationSessionUseCase` + `CloseCollaborationSessionUseCase` in `use-cases/project/` вЂ” `onConnect`/`onDisconnect` hooks delegate to these; enables domain unit testing independent of Hocuspocus runtime
   f. Refactor `SaveDocumentContentUseCase` (add session guard) вЂ” TDD
   g. Refactor `DeleteFileUseCase` (add session guard) вЂ” TDD
3. **Infrastructure layer**
   a. `PrismaCollaborationSessionRepository` + integration test
   b. Extend `PrismaDocumentRepository` with `findByYjsStateId`
4. **`apps/api` changes**
   a. Delete `hocuspocus-persistence.ts` plugin stub
   b. Add `internal-server.ts` (second Fastify instance bound to 127.0.0.1:4001)
   c. Add `GET /internal/collab/auth` route with UUID v4 schema validation + tests
   d. Add `ASCIIDOCOLLAB_COLLAB_INTERNAL_PORT` to convict config schema
5. **`apps/collab` new app**
   a. Package scaffold (`package.json`, `tsconfig.json`)
   b. Config schema
   c. `persistence.ts` extension (moved + extended with file write-back + bootstrap)
   d. `auth-hook.ts` extension
   e. `composition-root.ts`
   f. `server.ts` + `index.ts` with graceful shutdown
   g. Tests for extensions
