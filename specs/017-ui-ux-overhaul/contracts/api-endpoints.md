# API Contracts: UI/UX Overhaul

**Branch**: `017-ui-ux-overhaul` | **Date**: 2026-06-07

All endpoints require `credentials: 'include'` (session cookie). Admin endpoints additionally require the `isAdmin` flag on the session user.

---

## New Endpoints

### `GET /projects/:projectId/download`

Download entire project as a streaming ZIP archive.

**Auth**: Authenticated session; project member (any role).

**Path params**:
- `projectId` — UUID

**Response**:
- `200 OK` — streaming binary body
  - `Content-Type: application/zip`
  - `Content-Disposition: attachment; filename="<project-name>-<YYYY-MM-DD>.zip"`
  - Body: ZIP stream generated by archiver, no intermediate file written to disk
- `403 Forbidden` — not a project member
- `404 Not Found` — project not found

---

### `GET /projects/:projectId/files/:fileNodeId/download`

Download a single file from the project file tree.

**Auth**: Authenticated session; project member (any role).

**Path params**:
- `projectId` — UUID
- `fileNodeId` — UUID

**Response**:
- `200 OK`
  - `Content-Type`: original file MIME type
  - `Content-Disposition: attachment; filename="<original-filename>"`
  - Body: file content stream from `ProjectFileStore`
- `400 Bad Request` — `fileNodeId` is a folder, not a file
- `403 Forbidden` — not a project member
- `404 Not Found` — file node not found

### `GET /admin/audit-logs`

List audit log entries with optional filters and pagination.

**Auth**: Authenticated session; `isAdmin = true`.

**Query params** (all optional):
- `fromDate` — ISO 8601 date string (inclusive)
- `toDate` — ISO 8601 date string (inclusive)
- `userId` — UUID; filter by actor
- `actionType` — string; filter by action value
- `page` — integer ≥ 1 (default: 1)
- `limit` — integer 1–200 (default: 50)

**Response `200 OK`**:
```json
{
  "items": [
    {
      "id": "uuid",
      "userId": "uuid | null",
      "userDisplayName": "string | null",
      "projectId": "uuid | null",
      "action": "string",
      "resourceType": "string",
      "resourceId": "string",
      "timestamp": "ISO 8601",
      "metadata": {}
    }
  ],
  "total": 1234,
  "page": 1,
  "limit": 50
}
```

**Response `403 Forbidden`** — not an admin.

### `GET /admin/audit-logs/action-types`

Return the distinct set of `action` values recorded in the database. Used to populate the filter dropdown.

**Auth**: Authenticated session; `isAdmin = true`.

**Response `200 OK`**:
```json
{ "actionTypes": ["USER_LOGIN", "FILE_RENAMED", "PROJECT_CREATED", ...] }
```

## Modified Endpoints

### `PATCH /auth/me/profile` — add `avatarKey` and `appTheme`

Existing route; body schema extended.

**Added body fields**:
```json
{
  "avatarKey": "string | null",
  "appTheme": "\"light\" | \"dark\" | \"system\""
}
```

`avatarKey`: validated as `string | null`, max-length ≤ 50 characters. The server does not validate against a specific set of style keys — style enumeration is a frontend concern. Unknown keys are stored as-is and resolved to `DEFAULT_AVATAR_STYLE` at render time. `null` clears any saved preference.
`appTheme`: validated against `["light", "dark", "system"]`. Invalid value → `400 Bad Request`.

On success, the response includes the updated `appTheme` value. The caller (frontend) is responsible for writing the `asciidocollab-theme` browser cookie as a cache after a successful response.
