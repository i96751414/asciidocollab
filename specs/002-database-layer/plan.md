# Implementation Plan: Database Layer

**Branch**: `002-database-layer` | **Date**: 2026-05-27 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from `specs/002-database-layer/spec.md`

## Summary

Implement a PostgreSQL persistence layer for all 9 domain entities (User, Project, ProjectMember, FileNode, Document,
Image, Template, GitRepository, AuditLog) using Prisma. This includes a Prisma schema, generated client, concrete
implementations of all 9 repository interfaces, and integration tests against a real database via testcontainers.

## Technical Context

**Language/Version**: TypeScript 5.5+

**Primary Dependencies**:

- `@prisma/client` — generated type-safe database client
- `prisma` (dev) — schema management, migrations, client generation
- `testcontainers` (dev) — PostgreSQL container for integration tests
- `@asciidocollab/domain` (workspace) — repository interfaces to implement
- `@asciidocollab/shared` (workspace) — DTOs and shared types

**Storage**: PostgreSQL via Prisma ORM (generated client)

**Testing**:

- Jest + ts-jest (per existing monorepo config)
- testcontainers for PostgreSQL-backed integration tests
- Each Prisma repository tested against a real database instance
- Coverage threshold at 90% (per monorepo standards for non-domain packages)

**Target Platform**: Node.js 24.x (Active LTS "Krypton") — same as Phase 1

**Project Type**: Library packages within a pnpm monorepo (`packages/db` + `packages/infrastructure`)

**Performance Goals**:

- Prisma schema generation: < 10 seconds
- Full integration test suite: < 30 seconds (including container startup)
- Repository query latency: < 50ms p95 for simple CRUD (single-row queries)

**Constraints**:

- `packages/db` — depends only on `@prisma/client`; no monorepo internal imports
- `packages/infrastructure` — may import `domain`, `shared`, `db`; domain MUST NOT import infrastructure
- All 9 repository interfaces from Phase 1 MUST be fully implemented with Prisma
- TypeScript `strict: true` in every `tsconfig.json`
- Code coverage >= 90% (per monorepo standards)
- fresh-onion must pass in CI (no architectural violations)
- No `any` type in production code
- No `as` type casts in production code
- All UUID columns use native PostgreSQL `uuid` type via `@db.Uuid`

**Scale/Scope**: 9 Prisma repository implementations, ~15 source files, 9 integration test files, Prisma schema covering
9 tables with 3 enums
