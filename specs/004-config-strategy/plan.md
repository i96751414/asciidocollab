## Summary

files with environment variable overrides. All 30+ settings will be defined in a single schema, loaded from layered YAML
files (default â†’ environment-specific â†’ env var overrides), with type-safe access and sensitive field redaction.

## Technical Design

### Convict Schema

Single schema file defines all config fields with:

- `doc` â€” human-readable description (shows in YAML comments)
- `format` â€” type validation (integer, port, hostname, string, boolean, required-string)
- `default` â€” fallback value
- `env` â€” environment variable name for override
- `sensitive` â€” marks secrets for redaction

### YAML Loading Order

```
1. Load apps/api/config/default.yaml     (base defaults)
2. Load apps/api/config/{NODE_ENV}.yaml   (environment override)
3. Apply environment variable overrides   (highest priority)
```

### Sensitive Fields

Fields marked `sensitive: true` in the schema:

- `auth.session.secret`
- `auth.session.encryptionKey`
- `auth.smtp.password`
- `auth.sendgrid.apiKey`

These fields:

- MUST NOT have default values in YAML
- Are redacted in convict output (`convict.toString()` replaces with `[REDACTED]`)
- Are validated as `required-string` in production

### Migration Path

1. Create new config schema and loader
2. Create YAML files with current defaults
3. Update route handlers to use `app.config` instead of `process.env`
4. Remove `@fastify/env` dependency
5. Update `env.ts` â†’ `index.ts` (or remove entirely)
6. Run tests to verify behavior unchanged
